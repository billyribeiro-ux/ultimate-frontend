# Module 17 — Real-time: Exercises

These five exercises build on the core concepts from Module 17: Server-Sent Events, EventSource consumers, notification feeds, WebSocket communication, and presence systems in SvelteKit.

## Difficulty Progression

| # | Title | Difficulty | Time | Key Skills |
|---|-------|-----------|------|------------|
| 1 | SSE Endpoint | beginner | 10 min | ReadableStream, Server-Sent Events, SvelteKit streaming |
| 2 | EventSource Consumer | intermediate | 20 min | EventSource API, reactive state updates, reconnection |
| 3 | Notification Feed | advanced | 30 min | Multi-event SSE, notification types, dismissal state |
| 4 | WebSocket Echo Server | expert | 45 min | WebSocket upgrade, bidirectional messaging, connection lifecycle |
| 5 | Presence System | principal | 60 min | User tracking, heartbeats, online/offline states, cleanup |

## Prerequisites

- Lessons 17.1 through 17.7 completed
- Understanding of HTTP streaming and event-driven architecture
- `pnpm dev` running successfully

## How to Use

1. Read the exercise brief carefully
2. Attempt the solution without hints first
3. Use progressive hints only when genuinely stuck
4. Compare your solution to the provided one — multiple approaches are valid
5. Each exercise builds confidence for the next; do them in order
