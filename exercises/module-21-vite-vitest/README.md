# Module 21 — Vite & Vitest Internals: Exercises

These five exercises build on the core concepts from Module 21: Vite configuration, environment variables, the plugin system, HMR boundaries, Vitest fundamentals, mocking strategies, and snapshot testing.

## Difficulty Progression

| # | Title | Difficulty | Time | Key Skills |
|---|-------|-----------|------|------------|
| 1 | Environment Variable Security Audit | beginner | 15 min | $env/static/private, $env/static/public, Vite env exposure |
| 2 | Custom Vite Plugin — Build Timestamp | intermediate | 25 min | Vite plugin API, virtual modules, transform hooks |
| 3 | HMR Boundary Understanding | advanced | 35 min | HMR propagation, accept boundaries, state preservation |
| 4 | Mock Strategy for SvelteKit Load Functions | expert | 45 min | vi.mock, vi.fn, server load testing, dependency injection |
| 5 | Snapshot Testing Judgment Call | principal | 60 min | toMatchSnapshot, toMatchInlineSnapshot, when to use/avoid |

## Prerequisites

- Lessons 21.1 through 21.10 completed
- `pnpm dev` running successfully
- Familiarity with Vite config, Vitest setup, and SvelteKit load functions

## How to Use

1. Read the exercise brief carefully
2. Attempt the solution without hints first
3. Use progressive hints only when genuinely stuck
4. Compare your solution to the provided one — multiple approaches are valid
5. Each exercise builds confidence for the next; do them in order
